## Правила работы

### Обязательное обращение к MCP серверам перед началом работы
Перед началом любой задачи (и в режиме планирования, и в режиме редактирования) — **обязательно обратиться к двум MCP серверам**:
1. **Context7** — для получения актуальной документации и примеров кода по используемым библиотекам/технологиям
2. **Shopify Dev MCP** (`learn_shopify_api`) — для получения актуального контекста по Shopify API, Liquid, расширениям и другим релевантным Shopify-технологиям

Только после получения контекста от обоих серверов — приступать к работе.

**Исключение:** для задач не связанных с редактированием liquid code/css code/js code в проекте — **не использовать MCP серверы** и использовать модель **Haiku** модель.

### Запрет изменений на ветке production
Если текущая git ветка — `production`, **запрещено вносить любые изменения** в файлы проекта. Сначала нужно переключиться на feature-ветку (`npm run open --name=...`).

### Коммиты после каждой фичи
После завершения любой задачи (фича, фикс, рефакторинг) — **обязательно вызвать `/commit`**. Это единственный способ зафиксировать изменения — `/commit` сам делает git commit и Shopify Push. **Никогда не вызывать `shopify theme push` напрямую** — только через `/commit`. Не жди отдельной команды на коммит — вызывай `/commit` сразу по завершении работы.

### Запрет inline-стилей
Никогда не использовать inline-стили (`<style>` теги) в секциях, сниппетах и любых других Liquid-файлах. Если нужны стили — создать отдельный CSS-файл в папке `assets/` с именем, соответствующим файлу (например, для `sections/n-hero.liquid` → `assets/n-hero.css`), и подключить его стандартным способом:
```liquid
{{ 'n-hero.css' | asset_url | stylesheet_tag }}
```

### Откат коммитов с Shopify Push
Когда Павел просит откатиться до определённого коммита — после выполнения git-отката **обязательно сделать Shopify Push** (`/commit`)

### Работа со скриншотами сайта
Когда Павел прикладывает скриншот страницы сайта — убедиться, что Playwright MCP браузер открыт (если нет — открыть главную). Проверить, совпадает ли скриншот с текущей страницей. Если не совпадает — уточнить у Павла, с какой страницы скриншот, перейти туда и продолжить работу.

### Открытие сайта через Playwright MCP
Когда Павел просит открыть сайт — использовать **Playwright MCP**.
URL для открытия по умолчанию:
```
https://qureskincare.com/?_ab=0&_fd=0&_sc=1&preview_theme_id=<ID из файла .theme_ids/<branch-name>>
```

Перед открытием — определить имя текущей ветки (без префикса `feature/`), прочитать файл `.theme_ids/<branch-name>` и подставить его значение в `preview_theme_id`.
