## Project Overview

Shopify Liquid theme for Qure Skincare (store: `qure-skincare`). This is a hybrid architecture — the theme repo is a template/configuration layer, while compiled CSS/JS assets are hosted externally on `qureskincaredns.com`.

## Development Commands

```bash
npm run start        # Local dev server with hot reload and theme editor sync
npm run publish      # Push as unpublished theme to Shopify
npm run sync         # Sync local production branch with remote
npm run open --name=branch-name                        # Create feature branch from production
npm run push --name=branch-name --message="msg"        # Commit all changes and push feature branch (--message is optional, defaults to "push")
npm run close --name=branch-name                       # Delete feature branch locally and remotely
```

All commands use Shopify CLI under the hood. The store is `qure-skincare`.

## Git Workflow

- **Main branch:** `production`
- **Feature branches:** `feature/<name>` — branch from production, merge via PR
- Many commits are auto-generated "Update from Shopify" syncs from the admin theme editor

## Architecture

### Asset Strategy
Most CSS and JavaScript is **not in this repo**. Core styles and scripts are served from `qureskincaredns.com` CDN:
- `/assets/css/vendor.css`, `base.css`, `spacings.css`, `theme-fonts.css`
- `/assets/js/vendor.js` — also defines `loadScriptOnce(id, url)`, a deduplication utility used throughout snippets for lazy-loading scripts
- Section-specific CSS/JS paths on the CDN: `https://qureskincaredns.com/sections/new/<name>/<name>.css` and `.js`, or `sections/isolated/` for standalone sections

Local assets in `/assets/` are supplementary: analytics (`datalayer.js`), cart drawer, bundle selection, product page features, and third-party integration scripts.

`snippets/__render-sections-stack.liquid` handles per-template CSS injection from the CDN for specific page templates. `snippets/header-scripts.liquid` and `snippets/footer-scripts.liquid` load third-party integrations (GTM, Kameleoon A/B, Stamped.io reviews, Sezzle BNPL, Zendesk, PostHog, Skimlinks).

### Theming
CSS custom properties `--primary-color` and `--discount-color` are set in `theme.liquid` from theme settings. Bootstrap-style utility classes are used throughout (e.g., `mb-30`, `d-flex`, `fs-16`).

### Blocks (2 files)
Shopify theme app extension blocks in `/blocks/` using the `{% content_for 'block' %}` API. CSS loaded inline from CDN. Currently: `block-card.liquid` and `_block-card-item.liquid`.

### Sections (197 files)
Self-contained Liquid components with JSON schema blocks for Shopify admin customization. Naming prefixes:
- `n-` — core theme sections (header, footer, product header)
- `__endrock_` — third-party gamification/testimonial integrations
- `n-product-` — product page feature sections

### Snippets (151 files)
Reusable Liquid partials included via `{% render 'snippet-name' %}`.

### Templates (121 files)
JSON-based templates that compose sections. Market-specific overrides use the `.context.<market>.json` suffix pattern (e.g., `product.face-serum.context.israel.json`).

### Multi-Market
Templates have per-market overrides for: US, Australia, Europe, France, Germany, Israel, United Kingdom. Configured in `config/markets.json`.

## Key Files

- `layout/theme.liquid` — Main HTML shell, CDN resource loading, CSS variables, structured data
- `config/settings_schema.json` — Theme settings definition (colors, header sliders, menus)
- `config/settings_data.json` — Current settings values (auto-managed by Shopify admin)
- `.shopify/metafields.json` — Custom metafield schema definitions
- `locales/en.default.json` — English translation strings

## Important Conventions

- `config/settings_data.json` and `config/markets.json` are auto-generated by Shopify admin — avoid manual edits
- Section schemas use `t:` prefixed keys for i18n (e.g., `"label": "t:sections.header.name"`)
- Scripts use `loadScriptOnce(id, url)` from CDN `vendor.js` for deduped async script loading; call it inside `<script>` tags within snippets
- Use comments sparingly — only comment complex or non-obvious logic
- No build pipeline or transpilation — JavaScript and CSS in `/assets/` are authored directly
- No testing framework — changes are verified via `npm run start` local preview
